Еще один проект больше для демонстрации, чем для использования. Моя пока
заброшенная попытка сваять собственный автоматический "трейдер" на бирже,
торгующий через брокерскую консоль QUIK.

Десктопное ПО QUIK предоставляет возможность расширять свой функционал при
помощи написания скриптов Lua. Мне как-то было странно в 2022 году писать
что-либо заметное на Lua. Здесь более подходит, например, все тот же Python -
как по функциональности, так и по перспективам когда-нибудь припаять к
системе глубокое обучение (lol).

Однако встала задача как интегрировать QUIK с Питоном, он умеет только с Lua.
Готовых способов не нашлось, поэтому пришлось ваять собственную DLL на C++,
которая позволяет QUIK-у через Lua общаться с Питоном посредством REST на
localhost. Если все грамотно организовать, все работает достаточно шустро.

Python-часть проекта уже умеет делать потоковую обработку биржевых данных,
полученных из QUIK, и отдавать эти данные обратно в QUIK для построения
графиков прямо в его среде. Но в один момент я решил круто реорганизовать
свой код, все разломал, а вот обратно не собрал ввиду переезда в другой город
и прочих семейных заморочек. Поэтому пока все встало, но как образчик моего
Python-кода наверное сойдет.

Каталоги проекта:
LuaIndicators - подкаталог из ПО QUIK с Lua-скриптами для обмена с Python
microhttp - DLL для Lua, добавляющая возможность слать из него http-запросы
microtrader - незавершенный пока проект на Python
